# 모델 실험 여정 — E1 에서 앙상블까지

실제 실험 로그(`experiments.md`, `logs/*.meta.json`, `results/ablation_weights.json`) 를 기반으로 작성했다. 숫자는 전부 로그에서 그대로 옮겨온 값이다.

## 한 줄 요약

ResNet50 freeze(고정) 로 시작한 베이스라인이 0.4042 였다. 여기서 fine-tune, 아키텍처 교체, Transformer 전환, 자체 encoder 설계, Knowledge Distillation, 4 모델 앙상블을 차례로 거쳐 최종 val 0.8692 에 도달했다.

데이터에는 본래 라벨 불확실성이 존재했다. 3 인 라벨러의 **다수결 일치율은 0.9045, 전원 일치율은 0.6108** 이었다. 즉 사람들끼리도 39% 구간에서는 의견이 갈리는 과제였다. 최종 앙상블 0.8692 는 이 합의 수준 (0.85~0.90 권역) 에 근접한 결과였다.

**문서 주석**: 실험 번호 (E1~E9) 는 실제 학습 로그 순서를 따랐다. 본문에서 E9(KD) 가 E8(앙상블) 보다 먼저 나오는 것은 앙상블이 E9 결과까지 포함해 최종 탐색을 돌렸기 때문이다.

## E1. ResNet50 Freeze 베이스라인 — val 0.4042

가장 가벼운 설정부터 시작했다. 데이터·전처리·평가 파이프라인이 끊김 없이 도는지 먼저 확인하는 목적이 컸다.

| 설정 | 값 |
|---|---|
| 모델 | ResNet50 (ImageNet pretrained) + FC head |
| Freeze | backbone 전체 고정, head 만 학습 |
| bs / lr / epoch | 128 / 1e-3 / 10 |
| Face crop | 사용 안 함 |
| Augmentation | 사용 안 함 |

결과는 val 0.4042, train 0.4868 로 gap 0.08 의 underfit 상태였다.

낮게 나온 원인이 셋이었다.
1. 입력이 원본 이미지 통째라서 배경 노이즈가 학습 신호를 오염시킴
2. backbone 을 고정했으니 감정 표정의 세부 feature 를 새로 배울 수 없음
3. 4-class random 기준이 0.25 인데 거기서 +15%p 올라간 수준

얻은 것은 명확했다. freeze 만으로 안 된다. fine-tune 이 필요하고 face crop 도 같이 붙어야 한다.

## E2. ResNet50 ft(conv5) + Crop + Aug — val 0.7733

E1 에서 드러난 세 가지 문제를 한 번에 해결하려고 묶어서 돌렸다.

| 변경 | 근거 |
|---|---|
| conv5 unfreeze | freeze 한계를 벗어나 마지막 블록부터 학습 허용 |
| Face crop (annot_A.boxes) | 배경 제거해서 얼굴 영역만 학습 |
| Augmentation (flip, rot±10, colorjitter) | 7,196 장 규모에서 overfit 방어 |
| bs 128 → 32 | gradient 업데이트를 더 세밀하게 |
| lr 1e-3 → 1e-4 | fine-tune 단계에 맞게 축소 |
| label smoothing 0.1 | 모델이 너무 자신하지 않게 보정 |

결과는 val 0.7733, train 0.988 로 gap 0.21. +37.0%p 상승이 한 번에 나왔다. face crop, conv5 unfreeze, augmentation 셋의 조합이 상승폭의 대부분을 만들었다. 다만 gap 0.21 은 overfit 의 신호였다.

## E3. AdamW + CosineWarmup + StrongAug + Dropout 0.5 — val 0.7617

E2 의 overfit 을 잡으려고 정규화를 전면적으로 강화했다.

| 강화 항목 | 설정 |
|---|---|
| Optimizer | AdamW (weight_decay=1e-4) |
| LR schedule | cosine + warmup 3 epoch |
| Augmentation | rot±15, zoom 0.1, translation 0.05, colorjitter 0.15 |
| Dropout | 0.5 |
| Label smoothing | 0.2 |
| Monitor | val_loss (val_acc 대신) |

gap 은 E2 의 0.21 에서 0.20 으로 거의 그대로였다 (final train 0.9624, best val 0.7617). 정규화는 강화했는데 overfit 억제 효과는 미미했고, val_acc 는 0.7617 로 E2 대비 -1.2%p 오히려 떨어졌다.

나중에 원인을 짚어보니 monitor 를 val_loss 로 바꾼 선택이 컸다. label_smoothing 을 0.2 까지 올리면 loss 계산 기준이 바뀌면서, 정확도 기준의 best checkpoint 를 놓치는 상황이 생긴다. 이후 실험부터는 monitor 를 val_accuracy 로 되돌리고 label_smoothing 도 0.1 로 내렸다. 이 실험에서 얻은 것은 성능 향상이 아니라 "어떤 방향으로 튜닝하면 안 되는지" 라는 부정적 정보였다.

## E4. EfficientNet-B0 ft(block6) — val 0.7958

ResNet50 이 0.76~0.77 에서 정체했다. regularization 을 더 조여도 크게 바뀌지 않아서 아키텍처 자체를 바꿔봤다.

| 근거 | 설명 |
|---|---|
| ResNet50 한계 | 23M 파라미터 중 상당수가 우리 과제에 재사용되지 않음 |
| EfficientNet-B0 | 5.3M 파라미터로 ImageNet 에서 ResNet50 과 거의 동등 |
| Compound scaling | width·depth·resolution 을 균형 있게 조정한 구조 |
| Fine-tune 범위 | block6 (마지막 단계) |
| Augmentation | 중간 강도로 되돌림 (rot 12, zoom 0.05). E3 의 strong aug 는 over-smoothing 이 의심됨 |

val 0.7958, gap 0.15. E3 와 같은 gap 을 유지하면서 val_acc 는 +2.25%p 올랐다. 정규화 조이기보다 아키텍처 교체가 이 상황에서 더 빠른 효과를 냈다는 뜻이다. 이 시점에서 CNN 트랙의 천장이 0.80 근처라는 점도 같이 드러났고, 다음은 다른 계열로 가야 했다.

## E5. ViT-B/16 2-stage Fine-tune — val 0.8458

CNN 의 천장을 넘기 위해 Transformer 로 옮겼다. 프로젝트 가이드 PDF p.26~28 도 ViT-B/16 을 지정했다.

| 근거 | 설명 |
|---|---|
| 전역 attention | 얼굴 전체의 관계를 한 번에 본다. CNN 이 놓치던 장거리 관계를 포착 |
| ImageNet21k pretrain | 이미지 전반의 supervised pretrain 을 전이학습 |
| 2-stage fine-tune | Phase A: head only (5 ep, lr 1e-3) → Phase B: backbone 전체 unfreeze (15 ep, lr 5e-5) |
| bf16 mixed precision | 메모리·속도 동시 이득 |
| cosine warmup 2 | 안정 수렴 |

val 0.8458, train 0.9826, gap 0.14, val_f1 0.8470. CNN 트랙 대비 +4.99%p 상승 (학습 best 기준, 재평가 실측은 아래 "재현 실측" 표 참조). 얼굴 표정의 미묘한 관계가 중요한 이 과제에서 Transformer 가 구조적으로 유리하다는 점이 확인됐다. 단일 모델 중 가장 높은 값이라, 제출에서도 앙상블이 깨졌을 때 쓰는 fallback 경로로 이 checkpoint 를 쓰기로 했다.

여기서 한 번 더 끌어올리려면 같은 계열을 또 학습하기보다는 "다른 family" 를 붙여 다양성을 만드는 쪽이 효율적이었다.

## E6. SigLIP + yua-encoder Linear Probe — val 0.8192

팀에서 자체 설계한 yua-encoder(SigLIP projection wrapper) 를 붙였다. 이유는 둘이었다.

1. 팀 고유 설계라는 점에서 창의성 점수에 직결
2. ViT 와 다른 pretrain paradigm (SigLIP 은 image-text contrastive). 앙상블용 다양성 재료

| 구조 | 값 |
|---|---|
| Backbone | SigLIP-Base patch16-384 (WebLI contrastive pretrain) |
| Freeze | backbone 전체 freeze (linear probe, 선형 프로빙) |
| PixelShuffle r=4 | 576 → 36 tokens (팀 원작 설계) |
| Projection MLP | 25M trainable |
| Mean pool + Linear(4) | 최종 분류 head |
| img_size | 384 |
| class weight | auto (역빈도 기반) |

결과는 val 0.8192. E5 보다 -2.66%p 낮다. linear probe 라 학습 capacity 가 제한적이어서 예상 범위였다. 단독 성능은 ViT 에 못 미쳤지만 실수 패턴 자체가 달라서 앙상블 재료로는 충분했고, 다음 단계에서 backbone 일부를 풀고 Knowledge Distillation 을 얹는 방향으로 이어졌다.

## Knowledge Distillation 이 뭔지 — 쉽게

한 문장으로 정리하면 이렇다.

> 시험을 먼저 본 "선생님(teacher)" 이 학생(student) 에게 정답만 주는 게 아니라, "이건 A 가 맞는데 B 도 좀 비슷해, C 는 아니야" 같은 확신도 분포를 통째로 건네주는 것.

왜 효과가 있냐면, 기존 라벨은 `[1, 0, 0, 0]` 같은 one-hot 이라 정답은 알려주지만 나머지 선택지 사이의 관계는 전혀 가르쳐주지 못한다. 반면 선생님의 soft label 은 `[0.72, 0.03, 0.20, 0.05]` 처럼 분포 자체가 정보를 담고 있다. 예를 들어 "이 사진은 anger 지만 panic 으로도 꽤 볼 수 있는 애매한 경우" 같은 뉘앙스가 그대로 전달된다. 학생이 KL divergence 로 이 분포를 따라가도록 학습하면 단순 정답 학습보다 더 풍부한 표현을 얻는다.

우리 프로젝트에 도입한 이유는 셋이다.

- E6 SigLIP 의 linear probe 가 0.8192 에서 막혔다. capacity 가 부족한 것이 원인이었다.
- E2 + E4 + E5 앙상블이 이미 상당히 괜찮은 확신도 분포를 내고 있었다.
- 이 앙상블을 teacher 로 두고, E6 의 backbone 마지막 몇 블록을 풀어 student 로 다시 학습하면, capacity 는 열면서 앙상블의 판단 감각을 주입할 수 있다.

실제 결과로 같은 student 구조가 0.8192 에서 0.8383 으로 올랐다. +1.91%p.

수식은 이렇게 생겼다.

```
Total Loss = α · KL(student_soft ∥ teacher_soft) · T²
           + (1-α) · CrossEntropy(student, hard_label)
```

- T (temperature): softmax 를 얼마나 부드럽게 만들지 결정. 우리는 4
- α: teacher 비중. 우리는 0.7
- T² 곱하는 이유: gradient scale 을 맞추는 원 논문(Hinton 2015) 공식

## E9. SigLIP KD 실험 (Ablation 5 config) — val 0.8383

teacher 는 E2+E4+E5 의 softmax probs 를 미리 계산해서 `train_teacher_tsoff.npz` 로 저장해두고, student 는 E6 구조의 마지막 N block 만 unfreeze(해제) 했다.

| Name | T | α | unfreeze | val_acc | 비고 |
|---|:-:|:-:|:-:|---:|---|
| tsoff_T4_a07_uf0 | 4.0 | 0.7 | 0 | 0.8308 | baseline KD |
| tson_T4_a07_uf0 | 4.0 | 0.7 | 0 | 0.8292 | teacher softmax on. 소폭 하락 |
| tsoff_T2_a07_uf0 | 2.0 | 0.7 | 0 | 0.8267 | T 낮춤. 퇴행 |
| tsoff_T4_a07_uf2 | 4.0 | 0.7 | 2 | 0.8317 | last-2 block unfreeze |
| ce_only_control | 4.0 | 0.0 | 0 | 0.8267 | α=0. KD 끈 대조군 |
| tsoff_T4_a07_uf4 | 4.0 | 0.7 | 4 | **0.8383** | 최종 채택 |

세 가지를 확인했다.

1. unfreeze=4 가 가장 잘 나왔다. student 의 backbone capacity 를 열어줄수록 이득이 붙었다.
2. T=4 가 T=2 보다 좋았다. soft label 분포가 완만할수록 student 가 teacher 의 뉘앙스를 따라가기 쉬웠다.
3. α=0 (CE only) 이 0.8267, α=0.7 KD 가 0.8308. 다른 조건은 동일한 두 설정을 직접 비교해 KD 만의 고유 이득을 +0.41%p 로 분리했다.

E9 단독은 E5 ViT(0.8458) 보다 여전히 낮지만, 앙상블 재료로서 "다른 family 의 상보적 시선" 을 확보한 것이 핵심이었다.

## E8. 앙상블 탐색 (ensemble_search.py) — val 0.8692

각 모델의 실수 패턴이 서로 달라서, 개별 확신도를 잘 섞으면 상쇄 효과로 이득이 생긴다는 가정에서 출발했다.

### 비교한 5 가지 방법

| 방법 | 설명 | 선택 |
|---|---|:-:|
| Uniform voting | 모든 member 가중치 1/N | - |
| Temperature-Scaling voting | 각 member logits 에 T 적용 후 평균 | - |
| Weighted DE (Differential Evolution) | weight 공간 탐색으로 val 기준 최적 조합 | 선택 |
| Stacking LR OOF | out-of-fold logistic regression | - |
| Best single | 단일 모델 | - |

### DE 로 찾은 weight (val 1,200 장 기준)

| Member | 단독 val_acc | Weight | 역할 |
|---|---:|---:|---|
| E2 ResNet50 | 0.7758 | 0.041 | CNN 다양성 보조 |
| E4 EfficientNet-B0 | 0.7933 | 0.073 | CNN 다양성 보조 |
| E5 ViT-B/16 | 0.8383 | 0.523 | 주축 |
| E9 SigLIP KD | 0.8333 | 0.363 | 주축, 상보적 feature |

**수치 주석**: E5 ViT 의 단독 val_acc 가 본 표에서는 0.8383, E5 섹션에서는 0.8458 로 다르다. 전자는 DE search 과정에서 캐시된 softmax probs 로 argmax 재계산한 값이고, 후자는 학습 시 best checkpoint 의 실측값이다. 평가 프로토콜 차이이며 앙상블 계산에는 0.8383 기준을 사용한다. E9 의 0.8333 도 동일 방식이다.

Weight 합으로 보면 ViT 와 SigLIP-KD 가 88% 로 주축을 이루고, CNN 둘이 12% 로 다양성을 보강하는 구조다.

최종 지표: **val_acc 0.8692 / Macro F1 0.8697 / NLL 0.4593**.

## 앙상블 계산 — 실제로 어떻게 결정되나

val 에서 한 장을 예로 들면 이렇게 된다.

```python
# 각 member 의 softmax 확률 (shape: 4)
p_resnet = [0.78, 0.05, 0.09, 0.08]
p_effnet = [0.82, 0.03, 0.08, 0.07]
p_vit    = [0.91, 0.02, 0.03, 0.04]
p_siglip = [0.87, 0.03, 0.05, 0.05]

# weighted sum
p_ens = 0.041*p_resnet + 0.073*p_effnet + 0.523*p_vit + 0.363*p_siglip
#     = [0.8785, 0.0296, 0.0470, 0.0449]

argmax(p_ens) = 0 → "anger"
conf          = 0.8785
```

val 1,200 장 전부에 같은 식을 돌리면 1,043 장이 맞고, val_acc 0.8692 가 나온다.

## 추가 기법 총정리 — 9 카테고리 28 항목

중간중간 적용한 기법을 한 곳에 모았다. 튜터가 지적했듯 "왜 썼는지" 가 같이 있어야 발표에서 통한다. 괄호 안 한국어 표기는 보고서 기준 표준 용어.

### A. 데이터 · 전처리

| 기법 | 설명 | 이유 |
|---|---|---|
| EXIF orientation 정규화 | 사진 메타데이터 기준으로 자동 회전 보정 | 거꾸로 찍힌 사진을 바로잡기 |
| Face crop — 학습 | annot_A.boxes 좌표로 얼굴만 잘라 학습 | 배경 노이즈 제거 |
| MTCNN Face Detection — 추론 | 얼굴 영역 자동 검출 | bbox 없는 외부 이미지 대응 |
| 2-stage face pipeline | 학습은 annot, 추론은 MTCNN 으로 이중 구성 | 학습 조건 일관성 + 실사용 유연성 |
| Augmentation | flip · rot±10~15 · colorjitter · translation · zoom | 7,196 장 소규모 데이터의 overfit 방어 |

### B. 학습 전략

| 기법 | 설명 | 이유 |
|---|---|---|
| Freeze (고정) | 사전학습 가중치를 그대로 둠 | 초기엔 head 만 학습해 안정적으로 시작 |
| Unfreeze (해제) | 일부 층을 학습 가능하게 전환 | 우리 데이터에 맞게 세부 조정 |

### C. Optimizer · 학습률

| 기법 | 설명 | 이유 |
|---|---|---|

| 선택 | 근거 |

### H. Knowledge Distillation

| 기법 | 설명 | 이유 |
|---|---|---|
| Soft label (Teacher probs) | 확신도 분포 전체를 정답으로 사용 | 해설까지 포함된 답안지 |
| KD Temperature (T=4) | softmax 를 완만하게 만드는 상수 | teacher 의 뉘앙스를 student 가 따라잡기 쉽게 |
| α·KL + (1-α)·CE (α=0.7) | KD loss 와 hard label loss 혼합 | 두 정보를 동시에 학습 |

### I. 앙상블 · 추론

| 기법 | 설명 | 이유 |
|---|---|---|
| Soft voting | 각 member 의 확률을 가중평균 | 서로 다른 실수 패턴 상쇄 |
| Differential Evolution weight search | val 기준으로 최적 weight 조합 탐색 | uniform 대비 +3%p |
| Member temperature calibration | 각 member logit 에 별도 T 적용 (ResNet 1.03, EffNet 1.06, ViT 1.07, SigLIP-KD 3.32) | member 간 confidence 스케일 평준화 |
| Voting-level TS | ensemble_search.py 가 비교한 후보 방법 중 하나 | 최종 voting 방법 선정의 비교군 |
| TTA (5-crop · multi-scale · hflip) | 추론 시 이미지를 변형해 여러 번 예측 후 평균 | 한 장을 열 장처럼 활용해 안정성 향상 |

### Temperature Scaling 이 세 군데에서 나오는 이유

같은 이름이 카테고리 H, I 에 겹쳐서 나오는데, 실제 역할은 서로 다르다.

- **H · KD Temperature (T=4)**: 학습 단계에서 teacher soft target 을 부드럽게 만들어 student 가 따라가기 쉽게 한다.
- **I · Member calibration (T 값 각자 다름)**: 앙상블 때 member 별 확신도 스케일이 맞지 않아서, 각자 자기 logits 에 별도 T 를 곱해 평준화한다. SigLIP-KD 가 T=3.32 로 특히 큰 이유는 KD 학습 자체 T=4 의 영향으로 출력 분포가 평탄해진 것을 다시 날카롭게 조정한 것이다.
- **I · Voting-level TS**: 여러 앙상블 방법 중 하나의 후보. `ensemble_search.py` 가 DE, stacking 등과 함께 비교한 기법이다.

## 전체 스토리 — 왜 계속 바꾸다 앙상블로 갔나

1. E1~E4 에서 CNN 트랙의 한계를 확인했다. 파라미터, 아키텍처 크기, 정규화를 다 건드렸지만 0.80 근처가 벽이었다.
2. E5 에서 Transformer 로 옮겨 그 벽을 넘었다 (0.80 → 0.85).
3. E6 에서 SigLIP 으로 다른 family 를 확보했다 (단독 0.82, 앙상블 재료).
4. E9 에서 Knowledge Distillation 으로 student 를 강화했다 (0.82 → 0.84). 다양성과 성능을 같이 올렸다.
5. E8 에서 4 family 의 서로 다른 실수 패턴을 soft voting 으로 상쇄했다 (0.8692).

단일 모델 천장이 0.85 근처였고, 사람 합의 수준 (다수결 0.9045, 전원 0.6108) 에 근접하려면 서로 다른 inductive bias 를 가진 모델끼리 협업이 필요했다. Knowledge Distillation 이 다양성 확보와 성능 향상을 동시에 해주는 중간 다리 역할을 했다.

## 재현 실측 — 학습 best vs 추론 재평가

위 표의 val_acc 수치들이 **학습 로그에 기록된 best checkpoint 값** 과 정확히 맞는지 확인하기 위해, val 1,200 장 전체에 대해 전 모델을 다시 돌려 재계산했다. cache 가 있는 4 개 (E2/E4/E5/E9) 는 `results/ensemble_cache_phase2/*.npz` 를 그대로 썼고, 없는 4 개 (E1/E3/E6/Ensemble) 는 `predict.py` 파이프라인으로 재추론했다.

| Model | 학습 best (문서) | 재평가 실측 | Δ | 평가 조건 |
|---|---:|---:|---:|---|
| E1 ResNet50 frozen | 0.4042 | 0.3792 | -2.5%p | **crop off** 재추론 (학습 조건 일치) |
| E2 ResNet50 ft | 0.7733 | 0.7758 | +0.25%p | cache (학습 bbox crop) |
| E3 AdamW | 0.7617 | 0.7375 | -2.4%p | **MTCNN** 재추론 |
| E4 EfficientNet | 0.7958 | 0.7933 | -0.25%p | cache (학습 bbox crop) |
| E5 ViT-B/16 | 0.8458 | 0.8383 | -0.75%p | cache (argmax 재계산) |
| E6 SigLIP LP | 0.8192 | 0.7942 | -2.5%p | **MTCNN** 재추론 |
| E9 SigLIP KD | 0.8383 | 0.8333 | -0.5%p | cache |
| Ensemble FULL | 0.8692 | 0.8408 | -2.84%p | **MTCNN** 실추론 |

세 가지 범주로 해석된다.

1. **Cache 기반 4 개 (E2, E4, E5, E9)** — 거의 일치. 문서 숫자가 학습 로그와 정확히 맞는다는 증거.
2. **MTCNN 재추론 3 개 (E3, E6, Ensemble)** — 2~3%p 낮게 나왔다. 학습 시 annot_A.boxes 로 crop 한 얼굴 영역과, 추론 시 MTCNN 이 자동 검출한 영역이 완전히 일치하지 않는 것이 원인이다. README 의 "E8 bbox 0.8600 vs E8 MTCNN 0.8442 (-1.6%p) drift" 와 동일한 현상으로, 외부 테스트 환경의 실사용 성능은 이쪽에 더 가깝다고 봐야 한다.
3. **E1 crop off 재추론 (-2.5%p)** — MTCNN 이 아니라 아예 crop 을 끈 조건. 학습 파이프라인의 TF 내부 resize/normalize 와 `predict.py` 의 PIL resize 사이 미세 전처리 차이가 원인으로 보인다. 학습 로그 수치 (0.4042) 가 실제로 더 정확하다.

정리: 위쪽 표의 val_acc (0.8692 등) 는 **학습/검증 파이프라인 내부 기준** 의 정직한 수치고, MTCNN 만 쓸 수 있는 실전 환경에서는 앙상블 기준 **0.8408 정도** 를 기대하는 것이 타당하다. 제출 시 두 축을 함께 공개한다.
